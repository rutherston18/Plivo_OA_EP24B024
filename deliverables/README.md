# Deliverables — 2,000-step LLM speedrun (best run)

Everything needed to **reproduce, score, and understand** the best model.
Track: **byte-level language model**, scored by **bits-per-byte (bpb)** on held-out text.

**Best result: dev bpb `1.8225`** at **1,884,160 params** / 2000 steps (baseline `2.3718`, **−23.2%**),
within all constraints (≤2000 steps, ≤2M params, CPU-only, pure PyTorch, lossless tokenizer).

## Reproduce the score (fast, ~10 s)

Requires PyTorch (CPU is fine). We used conda env `plivo_env` with `torch 2.13.0+cpu`, `numpy 2.0.2`.

```bash
cd code
python evaluate.py --checkpoint ../ckpt.pt --text_file ../data/dev_eval.txt
# -> {"bpb": 1.8225, "n_params": 1884160, "steps": 2000, ...}
```

The tokenizer (`bpe.json`) loads automatically from `code/`, and the model config is stored
inside `ckpt.pt`, so the scorer rebuilds the exact architecture (including the n-gram tables).

## Retrain the best model from scratch (~10 min on CPU)

Point `--data` at the provided `train_corpus.txt` (not shipped here to keep the folder small):

```bash
cd code
python train.py \
  --data /path/to/train_corpus.txt \
  --steps 2000 --optimizer muon \
  --arch transformer --n_layer 4 --n_embd 160 --n_head 4 \
  --mlp_act relu2 --mlp_mult 4 --tie_weights \
  --ngram_orders 2,3 --ngram_buckets 1536 \
  --eval_every 200 --out ckpt.pt
```

This trains the E6 configuration (best run). The first run BPE-encodes the corpus once and
caches token ids next to the script to speed up subsequent runs.

To reproduce the whole ladder E1→E6, see the exact per-experiment commands and configs in
`RUNLOG.md` and the appendix of `SUMMARY.html`.

## Regenerate the report

`SUMMARY.html` is generated by `code/make_summary.py`, which reads the per-run archives in
the **main project repo** (`experiments/` + `baseline_results/`). Run it from the working
repo (`submission/`) rather than this standalone folder, where those archives aren't shipped.

## Notes on constraints

- **Pure PyTorch / numpy / stdlib only** — the tokenizer (byte-BPE), the Muon optimizer, and
  the SSM block are all reimplemented from scratch; no tiktoken / rustbpe / flash-attn / custom kernels.
- **Lossless tokenizer** — the byte-level BPE round-trips every byte (verified in the scorer).
- **≤ 2M params / ≤ 2000 steps / CPU** — the shipped model is 1,884,160 params, trained for 2000 steps on CPU.
